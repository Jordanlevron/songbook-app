- Keep responses short and direct. Don't narrate every step.
- Don't end responses with long "here's what I did" summaries — the user can see the diffs.
- Don't ask for confirmation before making straightforward fixes — just do them.
- Don't process issues one at a time if multiple are visible — address all of them together.
- Do flag when something requires a decision (visual design, destructive operation, etc.).

**Why:** User works efficiently and doesn't need hand-holding. Long narration wastes time.

**How to apply:** Bias toward action. Short updates at key moments. Save explanations for when something is genuinely surprising or requires a decision.
